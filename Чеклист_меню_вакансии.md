# Чек-лист тестирования: Скачивание меню и вакансии

## 1. Подготовительный этап

Тестовое окружение: Проверить функциональность скачивания и страницы вакансий в нескольких браузерах (Chrome, Firefox, Safari, Edge) и на разных устройствах (десктоп, планшет, мобильный).

Тестовые сценарии: Подготовить проверку всех элементов:

- [ ] Страница «Скачать меню»: 4 карточки PDF-файлов.
- [ ] Страница «Вакансии»: форма отклика с полями ФИО, E-mail, Номер телефона, выпадающий список должностей, сопроводительное письмо, reCAPTCHA и кнопка «Готов работать».

## 2. Страница «Скачать меню»

### 2.1. Карточка «Меню Italiano»

- [ ] Клик на карточку инициирует скачивание PDF или открывает файл в новой вкладке.
- [ ] Скачанный PDF открывается без ошибок и повреждений.
- [ ] Содержимое PDF соответствует ожидаемому (меню итальянской кухни).
- [ ] Имя скачиваемого файла читаемо и понятно (не набор символов).

### 2.2. Карточка «Винная карта»

- [ ] Клик на карточку инициирует скачивание PDF или открывает файл в новой вкладке.
- [ ] Скачанный PDF открывается без ошибок и повреждений.
- [ ] Содержимое PDF соответствует ожидаемому (винная карта).
- [ ] Имя скачиваемого файла читаемо и понятно.

### 2.3. Карточка «Банкетное меню»

- [ ] Клик на карточку инициирует скачивание PDF или открывает файл в новой вкладке.
- [ ] Скачанный PDF открывается без ошибок и повреждений.
- [ ] Содержимое PDF соответствует ожидаемому (банкетное меню).
- [ ] Имя скачиваемого файла читаемо и понятно.

### 2.4. Карточка «Детское меню»

- [ ] Клик на карточку инициирует скачивание PDF или открывает файл в новой вкладке.
- [ ] Скачанный PDF открывается без ошибок и повреждений.
- [ ] Содержимое PDF соответствует ожидаемому (детское меню).
- [ ] Имя скачиваемого файла читаемо и понятно.

### 2.5. Общие проверки скачивания

- [ ] Повторное скачивание одного и того же файла работает корректно (нет ошибки при второй попытке).
- [ ] При медленном соединении файл скачивается полностью, без обрыва.
- [ ] Размер скачанных PDF-файлов соответствует ожидаемому (файл не пустой и не обрезанный).
- [ ] На мобильных устройствах PDF открывается во встроенном просмотрщике или предлагается выбор приложения для открытия.

## 3. Страница «Вакансии» — Форма отклика

### 3.1. Отображение формы

- [ ] Форма содержит все поля: «ФИО», «E-mail», «Номер телефона», выпадающий список должностей, «Написать сопроводительное письмо», reCAPTCHA и кнопку «Готов работать».
- [ ] Все поля активны и доступны для ввода/выбора.
- [ ] Кнопка «Готов работать» отображается корректно.

### 3.2. Выпадающий список должностей

- [ ] Список раскрывается по клику на поле или стрелку.
- [ ] В списке присутствуют все три должности: «Бармен», «Повар», «Официант».
- [ ] В списке отсутствуют лишние или дублирующиеся позиции.
- [ ] После выбора должности выбранное значение отображается в поле.
- [ ] Список закрывается после выбора значения.
- [ ] Список закрывается при клике вне его области.

### 3.3. Валидация полей

- [ ] Отправка полностью пустой формы — отображается сообщение об ошибке для всех обязательных полей.
- [ ] Поле «ФИО»: отправка с пустым полем — отображается сообщение об ошибке.
- [ ] Поле «ФИО»: ввод одного символа (пограничное значение) — проверить поведение.
- [ ] Поле «E-mail»: ввод невалидного формата (например, «test», «test@», «@mail.ru») — отображается сообщение об ошибке.
- [ ] Поле «E-mail»: ввод валидного формата (например, «test@mail.ru») — ошибка не отображается.
- [ ] Поле «Номер телефона»: ввод букв или спецсимволов — отображается сообщение об ошибке.
- [ ] Поле «Номер телефона»: ввод слишком короткого номера (менее 10 цифр) — отображается сообщение об ошибке.
- [ ] Поле «Номер телефона»: ввод валидного номера — ошибка не отображается.
- [ ] Поле должности: отправка формы без выбора должности — отображается сообщение об ошибке (если поле обязательное).
- [ ] Поле «Сопроводительное письмо»: проверить, является ли поле обязательным — отправка без заполнения.
- [ ] reCAPTCHA: отправка формы без прохождения проверки — кнопка «Готов работать» неактивна или отображается сообщение об ошибке.

### 3.4. Успешная отправка

- [ ] Заполнение всех полей валидными данными, прохождение reCAPTCHA и нажатие «Готов работать» — форма успешно отправляется.
- [ ] После отправки отображается сообщение об успехе.
- [ ] После успешной отправки поля формы очищаются или форма переходит в состояние «успех».
- [ ] Повторная отправка (двойной клик на «Готов работать») не создаёт дублирующихся заявок.

### 3.5. Общие проверки

- [ ] При потере интернет-соединения в момент отправки — отображается понятное сообщение об ошибке.
- [ ] На мобильных устройствах форма отображается корректно, все поля доступны для ввода.
- [ ] Клавиатура на мобильных устройствах переключается в нужный режим: цифровая для поля «Номер телефона», email-раскладка для поля «E-mail».
