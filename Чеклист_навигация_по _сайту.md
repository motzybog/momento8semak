# Чек-лист тестирования: Навигация по сайту «Momento»

## 1. Подготовительный этап

Тестовое окружение: Проверить корректность отображения и работу навигации в нескольких браузерах (Chrome, Firefox, Safari, Edge) и на разных устройствах (десктоп, планшет, мобильный).

Тестовые сценарии: Подготовить проверку всех элементов навигации:

- [ ] Хедер (шапка сайта) с логотипом и пунктами меню.
- [ ] Все пункты навигационного меню: «Главная», «О нас», «Скачать меню», «Контакты», «Вакансии», «Парковка».
- [ ] Кнопки на главной странице: «Забронировать стол», «Получить банкетное предложение».
- [ ] Иконка поиска в хедере.

---

## 2. Хедер (шапка сайта)

### 2.1. Логотип

- [ ] Клик по логотипу выполняет переход на главную страницу.
- [ ] При наведении на логотип курсор меняется на pointer (рука).

### 2.2. Навигационное меню — общие проверки

- [ ] Все пункты меню присутствуют в хедере: «Главная», «О нас», «Скачать меню», «Контакты», «Вакансии», «Парковка».
- [ ] Активный пункт меню визуально выделен (подчёркивание или другой стиль) на соответствующей странице.
- [ ] При наведении на пункт меню отображается визуальный эффект (hover-состояние).
- [ ] Хедер фиксирован (остаётся видимым при прокрутке страницы вниз) — если предусмотрено дизайном.

---

## 3. Проверка пунктов навигационного меню

### 3.1. «Главная»

- [ ] Клик на пункт «Главная» выполняет переход на главную страницу сайта.
- [ ] Пункт «Главная» выделен как активный при нахождении на главной странице.
- [ ] Переход на «Главную» с любой другой страницы работает корректно.

### 3.2. «О нас»

- [ ] Клик на пункт «О нас» выполняет переход на соответствующую страницу.
- [ ] Страница загружается без ошибок (нет 404, нет пустого контента).
- [ ] Пункт «О нас» выделен как активный при нахождении на данной странице.

### 3.3. «Скачать меню»

- [ ] Клик на пункт «Скачать меню» выполняет переход на страницу загрузки меню.
- [ ] Пункт «Скачать меню» выделен как активный при нахождении на данной странице.

### 3.4. «Контакты»

- [ ] Клик на пункт «Контакты» выполняет переход на страницу контактов.
- [ ] Пункт «Контакты» выделен как активный при нахождении на данной странице.

### 3.5. «Вакансии»

- [ ] Клик на пункт «Вакансии» выполняет переход на соответствующую страницу.
- [ ] Страница загружается без ошибок (нет 404).
- [ ] Пункт «Вакансии» выделен как активный при нахождении на данной странице.

### 3.6. «Парковка»

- [ ] Клик на пункт «Парковка» выполняет переход на соответствующую страницу.
- [ ] Страница загружается без ошибок (нет 404).
- [ ] Пункт «Парковка» выделен как активный при нахождении на данной странице.

---

## 4. Кнопки действий на главной странице

### 4.1. «Забронировать стол»

- [ ] Кнопка отображается на главном экране (Hero-блок).
- [ ] Кнопка кликабельна и выполняет переход на форму бронирования или открывает модальное окно.
- [ ] При наведении на кнопку отображается hover-эффект.

### 4.2. «Получить банкетное предложение»

- [ ] Кнопка отображается рядом с кнопкой бронирования.
- [ ] Кнопка кликабельна и выполняет переход на соответствующую страницу или форму.
- [ ] При наведении на кнопку отображается hover-эффект.

---

## 5. Иконка поиска

- [ ] Иконка поиска (лупа) отображается в хедере на страницах «Скачать меню», «Контакты», «Вакансии», «Парковка».
- [ ] Клик на иконку открывает строку поиска или соответствующий интерфейс.
- [ ] Закрытие строки поиска работает корректно (кнопка закрытия или нажатие Esc).

---

## 6. Адаптивность навигации

- [ ] На мобильных устройствах (ширина до 768px) меню переключается в режим «бургер» (hamburger menu) или аналогичный адаптивный вид.
- [ ] Кнопка «бургер» открывает мобильное меню со всеми пунктами навигации.
- [ ] Все пункты меню кликабельны и работают корректно в мобильном виде.
- [ ] После выбора пункта меню на мобильном устройстве меню автоматически закрывается.
- [ ] На планшете (ширина 768–1024px) навигация отображается корректно без наложений и обрезания текста.

---

## 7. Общие проверки навигации

- [ ] Переход по кнопке «Назад» браузера возвращает пользователя на предыдущую страницу без ошибок.
- [ ] URL-адреса страниц соответствуют ожидаемым и читаемы (ЧПУ).
- [ ] Нет «битых» ссылок (404 Not Found) ни в одном пункте меню.
- [ ] Открытие любой страницы сайта по прямой ссылке (без перехода через меню) работает корректно.
- [ ] Все переходы происходят в той же вкладке браузера, если не предусмотрено иное дизайном.
- [ ] Скорость перехода между страницами приемлема (не более 3 секунд при стандартном соединении).
