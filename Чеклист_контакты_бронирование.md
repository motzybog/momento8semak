# Чек-лист тестирования: Работа с контактной информацией и бронированием

## 1. Подготовительный этап

Тестовое окружение: Проверить работу форм и контактных элементов в нескольких браузерах (Chrome, Firefox, Safari, Edge) и на разных устройствах (десктоп, планшет, мобильный).

Тестовые сценарии: Подготовить проверку всех контактных и бронировочных элементов:

- [ ] Контактные данные на главной странице (адрес, телефон, время работы).
- [ ] Страница «Контакты»: блок с контактными данными и форма обратной связи.
- [ ] Кнопка «Забронировать стол» и соответствующая форма/модальное окно.
- [ ] Кнопка «Получить банкетное предложение» и соответствующая форма/страница.

---

## 2. Контактные данные на главной странице

- [ ] Адрес ресторана отображается корректно: «г. Челябинск, проспект Ленина, 89а».
- [ ] Время работы отображается корректно: «11:00 – 24:00».
- [ ] Номер телефона отображается корректно: «+7 922 018-99-99».
- [ ] На мобильных устройствах номер телефона является кликабельной ссылкой (tel:), инициирующей звонок.

---

## 3. Страница «Контакты»

### 3.1. Блок контактных данных

- [ ] Адрес отображается корректно: «проспект Ленина, 89а Челябинск».
- [ ] Телефон отображается корректно: «+7 922 018-99-99».
- [ ] E-mail отображается корректно: «momento.rest@yandex.ru».
- [ ] На мобильных устройствах телефон является кликабельной ссылкой (tel:), e-mail — ссылкой (mailto:).

### 3.2. Форма обратной связи «Написать нам»

#### Отображение формы

- [ ] Форма содержит все три поля: «Имя», «Введите ваш телефон», «Сообщение».
- [ ] Все поля активны и доступны для ввода.
- [ ] Кнопки «Отправить» и «Сбросить» отображаются корректно.

#### Валидация полей

- [ ] Отправка пустой формы — отображается сообщение об ошибке для обязательных полей.
- [ ] Поле «Телефон»: ввод текста вместо цифр — отображается сообщение об ошибке формата.
- [ ] Поле «Телефон»: ввод номера в неверном формате (например, слишком короткий) — отображается сообщение об ошибке.
- [ ] Поле «Имя»: ввод спецсимволов или цифр — проверить поведение (ошибка или допустимо).
- [ ] Поле «Сообщение»: проверить наличие ограничения на максимальное количество символов (пограничное значение).

#### Успешная отправка

- [ ] Заполнение всех полей валидными данными и нажатие «Отправить» — форма успешно отправляется.
- [ ] После отправки отображается сообщение об успехе (например, «Спасибо, мы свяжемся с вами»).
- [ ] После успешной отправки поля формы очищаются или форма скрывается.

#### Кнопка «Сбросить»

- [ ] Нажатие «Сбросить» при заполненных полях — все поля очищаются.
- [ ] Нажатие «Сбросить» при пустой форме — ошибок не возникает.

---

## 4. Бронирование стола

### 4.1. Открытие формы бронирования

- [ ] Клик на кнопку «Забронировать стол» открывает форму бронирования или модальное окно.
- [ ] Форма/модальное окно открывается без ошибок и полностью отображается на экране.
- [ ] На мобильных устройствах форма/модальное окно не выходит за границы экрана.
- [ ] Закрытие модального окна (крестик или клик вне области) работает корректно.

### 4.2. Поля формы бронирования

- [ ] Форма содержит все необходимые поля (имя, телефон, дата, время, количество гостей — согласно дизайну).
- [ ] Все поля активны и доступны для ввода/выбора.
- [ ] Поле выбора даты: нельзя выбрать прошедшую дату.
- [ ] Поле выбора времени: значения ограничены временем работы ресторана (11:00 – 24:00).

### 4.3. Валидация формы бронирования

- [ ] Отправка пустой формы — отображается сообщение об ошибке для обязательных полей.
- [ ] Поле «Телефон»: ввод невалидного формата — отображается сообщение об ошибке.
- [ ] Поле «Количество гостей»: ввод 0 или отрицательного числа — отображается сообщение об ошибке.
- [ ] Поле «Количество гостей»: ввод числа, превышающего вместимость (пограничное значение) — проверить поведение.

### 4.4. Успешное бронирование

- [ ] Заполнение всех полей валидными данными и подтверждение — бронирование успешно отправляется.
- [ ] После отправки отображается сообщение об успехе с деталями брони (дата, время, имя).
- [ ] Модальное окно закрывается или форма переходит в состояние «успех» после отправки.

## 5. Банкетное предложение

### 5.1. Открытие формы/страницы

- [ ] Клик на кнопку «Получить банкетное предложение» открывает соответствующую форму или страницу.
- [ ] Форма/страница загружается без ошибок.
- [ ] На мобильных устройствах форма отображается корректно.

### 5.2. Поля формы банкетного предложения

- [ ] Форма содержит все необходимые поля (имя, телефон, дата мероприятия, количество гостей, комментарий — согласно дизайну).
- [ ] Все поля активны и доступны для ввода.

### 5.3. Валидация и отправка

- [ ] Отправка пустой формы — отображается сообщение об ошибке для обязательных полей.
- [ ] Поле «Телефон»: ввод невалидного формата — отображается сообщение об ошибке.
- [ ] Заполнение всех полей валидными данными и отправка — запрос успешно отправляется.
- [ ] После отправки отображается сообщение об успехе.

---

## 6. Общие проверки

- [ ] Все контактные данные на сайте консистентны (адрес, телефон и время работы одинаковы на главной странице и на странице «Контакты»).
- [ ] Повторная отправка формы (двойной клик на «Отправить») не создаёт дублирующихся заявок.
- [ ] При потере интернет-соединения в момент отправки формы — отображается понятное сообщение об ошибке.
- [ ] Все формы защищены от спама (reCAPTCHA или аналог) — если предусмотрено дизайном.
